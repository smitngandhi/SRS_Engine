🏗️ Shared Infrastructure

These components are created in Week 1 and never duplicated. Every feature imports from here.

  • parse_pdf(bytes) → str
  • parse_text(bytes) → str

  • works on any SRS/PRD/notes
📤 csv_exporter.py
  • export_backlog_csv(stories) → bytes
  • Jira-compatible columns
  • Linear-compatible columns
  • Notion CSV format
  • Used by Feature 4 only

// API Endpoints
POST/analyze-requirements
GET/requirements-scorer
// Output JSON Shape
  • overall_score: 0–100
  • breakdown: {clarity, testability, completeness, consistency, compliance}
  • requirements[]: {id, text, issues[], severity, suggested_fix}
// Groq Constraint Notes
  • No tool calls — pure prompt → JSON
  • Each scorer agent: independent prompt, returns partial score JSON
  • aggregator merges all parallel outputs from session state
// Packages Needed
  • No new packages — pure ADK + Groq
  • All existing utils reused
  • Pydantic for output validation

// API Endpoints
POST/upgrade-srs
GET/srs-upgrader
// Smart Upgrade Logic
  • STRONG sections → passed through unchanged
  • WEAK sections → rewritten with existing content as context
  • MISSING sections → generated from scratch using domain + other sections
  • Changelog entry for every change
// gap_detector_agent prompt key
  • "Given these sections, for each IEEE 830 section rate it STRONG (complete, specific, testable) / WEAK (exists but vague or incomplete) / MISSING (not found)"
  • Returns JSON — drives which upgrade agents run
// New packages needed
  • pdfplumber (pip install pdfplumber)
  • python-docx — already installed
  • python-multipart — for UploadFile

// CSV Column Schema (Jira-compatible)
  • Epic Name
  • Story ID (PROJ-001)
  • Summary (user story title)
  • Description (full story + ACs)
  • Story Points
  • Priority (High/Medium/Low)
  • SRS Reference (§3.2.1)
  • Issue Type (Epic/Story/Task)
// Domain-Aware Story Tags
  • Healthcare → "AC: Must comply with IEC 62304 §X"
  • Automotive → ASIL tag on safety stories
  • Finance → "AC: Must pass PCI DSS audit"
  • Education → "AC: WCAG 2.1 AA compliant"
